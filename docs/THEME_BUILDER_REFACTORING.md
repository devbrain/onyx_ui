# Conio Theme Builder Refactoring

**Date**: 2025-10-30
**Status**: Proposed
**Impact**: 90% code reduction (520+ lines → 60 lines)

---

## Problem Statement

The current `backends/conio/src/conio_themes.hh` contains **520+ lines** of highly repetitive theme definitions:

- 4 themes × 130 lines each
- Manual specification of every color state (normal, hover, pressed, disabled)
- Same colors repeated 20+ times per theme
- Difficult to maintain and modify
- No leverage of Phase 4 smart defaults system

**Example Pain Point** (current code):
```cpp
// Norton Blue theme - 130 lines of repetition
theme.button.normal = {
    .font = conio_renderer::font{.bold = false, .underline = false, .reverse = false},
    .foreground = color{255, 255, 255},  // ❌ Repeated 20+ times
    .background = color{0, 0, 170}       // ❌ Repeated 20+ times
};
theme.button.hover = {
    .font = conio_renderer::font{.bold = true, .underline = false, .reverse = false},
    .foreground = color{255, 255, 0},    // ❌ Manual specification
    .background = color{0, 0, 255}       // ❌ No smart defaults
};
// ... 100+ more lines
```

---

## Proposed Solution

### Architecture: Palette + Builder + Smart Defaults

```
theme_palette (colors) → theme_builder (fluent API) → theme_defaults (auto-generation) → Complete Theme
```

**Key Components:**

1. **`theme_palette`** - Define colors once using hex notation
2. **`theme_builder`** - Fluent API for theme construction
3. **`theme_defaults::apply_defaults()`** - Auto-generate all states (Phase 4)

## Implementation Details

### theme_palette Structure

```cpp
struct theme_palette {
    color window_bg;   // Base background
    color text_fg;     // Base foreground
    color border;      // Accent/border color
    color hover_bg;    // Auto-generated or custom

    static theme_palette from_hex(
        std::uint32_t bg,
        std::uint32_t fg,
        std::uint32_t border,
        std::uint32_t hover = 0  // 0 = auto-generate
    );
};
```

### theme_builder API

```cpp
theme_builder::create(name, description)
    .with_palette(bg, fg, border)       // Required: 3 base colors
    .with_hover_color(hover)            // Optional: custom hover
    .with_borders(border_style)         // Optional: border style
    .with_button_padding(h, v)          // Optional: padding
    .build();                           // Apply smart defaults
```

### Smart Defaults Flow

```cpp
build() {
    // 1. Apply theme_defaults (Phase 4)
    theme_defaults::apply_defaults(m_theme);

    // Auto-generates from 3 base colors:
    // - button.normal (from window_bg + text_fg)
    // - button.hover (lighten bg 20%, bold font, accent fg)
    // - button.pressed (invert colors)
    // - button.disabled (darken fg 60%)
    // - label.text, label.background
    // - panel, menu, menu_item, menu_bar_item
    // - scrollbar track, thumb, arrows

    // 2. Set non-color defaults
    // - Mnemonic fonts (underlined)
    // - Menu spacing/padding
    // - Border draw flags

    return theme;
}
```
